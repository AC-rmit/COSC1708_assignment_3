Testing

For the database, testing purposes could not be completed in the time frame that we had due to difficulties and creating the database in the middle of Week 11. However, plans were made to test the database with the integration of an API by using coordinates extracted from the City of Melbourne by reverse engineering addresses using longitude and latitude extracted from the City of Melbourne's databases using Google Maps. We tested accuracy of the information by using tables from the City of Melbourne database and putting it through the actual Google map servers to test whether the address associated with the longitude and latitude coordinates were accurate enough to be displayed. For most part, it was accurate enough that we were confident that API and integration would accurately view a designated location. Also, with markers. Abby had discovered that using JavaScript code. We could set multiple markers around a position to view any nearby car parks within a certain radius. However, we will not be able to test that functionality in the time we have.

For the user interface, ideally, we would have been using a live iPhone with the app ready to be tested from start to finish. However, in the time frame that we had, we tested the user interface by going through Figma and a flow chart diagram of the process of logging in, creating an account and entering an address to find the ideal car park space. Simulations of inputs and button pressing were done in through Figma in order to test the app designâ€™s user-friendly capability. Some changes to colour schemes and various icons had to be made in order to make the app more readable.
